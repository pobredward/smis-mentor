# 모바일 앱 데이터 프리로드 구현 완료

## 🎯 구현 내용

모바일 앱에서 스플래시 화면 시 데이터를 프리로드(캐시 등록) 및 프리마운트하는 기능을 구현했습니다.

### 주요 기능

1. **앱 시작 시 자동 프리로드**
   - 로그인 완료 후 사용자의 활성 캠프 데이터를 자동으로 프리페칭
   - 스플래시 화면에서 프리로드 진행 상황 표시
   - 최소 1초 표시 후 페이드아웃 효과

2. **캠프 변경 시 프리로드**
   - 마이페이지에서 캠프 변경 시 새로운 캠프 데이터 프리페칭
   - 기존 캐시 무효화 → 사용자 데이터 업데이트 → 데이터 프리페칭 → WebView 프리로딩
   - 진행률 표시 (4단계: 캐시 정리, 캠프 변경, 데이터 로딩, WebView 프리로딩)

3. **프리페칭 대상 데이터**
   - 수업 자료 (템플릿, 사용자 자료, 섹션)
   - 업무 데이터 (Tasks)
   - 시간표 링크
   - 인솔표 링크
   - 반명단 (ST시트 학생 데이터)
   - 방명단 (ST시트 학생 데이터)
   - 교육 자료 링크
   - WebView 프리로딩 (구글 시트 문서들)

## 📁 수정된 파일

### 1. `/packages/mobile/src/context/AuthContext.tsx`
**변경 사항:**
- `triggerDataPrefetch()` 함수 추가
- `registerPrefetchTrigger()`, `unregisterPrefetchTrigger()` 싱글톤 패턴 구현
- 로그인 완료 시 자동으로 프리페칭 트리거 (500ms 딜레이)
- `hasPrefetchedRef`로 중복 프리페칭 방지

**주요 로직:**
```typescript
// 사용자 데이터 로드 후 프리페칭 트리거 (한 번만)
if (!hasPrefetchedRef.current && userRecord.activeJobExperienceId) {
  hasPrefetchedRef.current = true;
  logger.info('🚀 AuthContext: 로그인 완료, 데이터 프리페칭 트리거');
  
  setTimeout(() => {
    triggerDataPrefetch();
  }, 500);
}
```

### 2. `/packages/mobile/src/types/index.ts`

### 3. `/packages/mobile/App.tsx`
**변경 사항:**
- 프리페칭 트리거 등록 (`registerPrefetchTrigger`)
- 스플래시 화면 표시 상태 관리 (`showSplash`)
- 비로그인 상태에서는 스플래시 숨기기
- `SplashPrefetchScreen` 컴포넌트 추가

**주요 로직:**
```typescript
// 프리페칭 콜백 등록
const handlePrefetch = async () => {
  if (!userData?.activeJobExperienceId) return;
  
  logger.info('🚀 앱 시작: 데이터 프리페칭 시작');
  await prefetchCampData(userData.activeJobExperienceId);
};

registerPrefetchTrigger(handlePrefetch);
```

**UI 구성:**

- 로딩 인디케이터

- 진행률 바 (WebView 프리로드 진행 상황)
- 퍼센트 표시 (예: "5 / 12 (42%)")

### 5. `/packages/mobile/src/components/index.ts`
**변경 사항:**
- `SplashPrefetchScreen` export 추가

### 6. `/packages/mobile/src/context/CampTabContext.tsx`
**변경 사항:**
- `loadLastPreloadLinks()` 로직 개선
- 앱 재시작 시 링크만 복원하고, 프리로드는 AuthContext에서 트리거하도록 수정
- 불필요한 자동 프리로드 시작 제거

**개선 이유:**
- 이전: 앱 시작 시 무조건 프리로드 시작 → 비로그인 상태에서도 실행
- 개선: 로그인 완료 후에만 프리로드 시작 → 불필요한 네트워크 요청 방지

## 🔄 프리로드 플로우

### 1️⃣ 앱 시작 (로그인 상태)
```
앱 실행
  ↓
AuthContext: onAuthStateChanged (로그인 감지)
  ↓
사용자 데이터 로드 (getUserByEmail)
  ↓
500ms 딜레이 (UI 우선 렌더링)
  ↓
triggerDataPrefetch() 호출
  ↓
App.tsx: handlePrefetch 실행
  ↓
useCampDataPrefetch: prefetchCampData()
  ├── 수업 자료 프리페칭
  ├── 업무 데이터 프리페칭
  ├── 시간표 데이터 프리페칭
  ├── 인솔표 데이터 프리페칭
  ├── 반명단 데이터 프리페칭
  ├── 방명단 데이터 프리페칭
  ├── 교육 자료 데이터 프리페칭
  └── WebView 프리로딩 시작
       ↓
SplashPrefetchScreen: 진행 상황 표시
  ├── 진행률 바 업데이트
  └── 완료 시 페이드아웃
       ↓
메인 화면 표시
```

### 2️⃣ 캠프 변경 (마이페이지)
```
사용자가 캠프 선택
  ↓
ProfileScreen: handleJobCodeSelect()
  ↓
Step 1: invalidateCampData() - 기존 캐시 무효화 (0% → 15%)
  ↓
Step 2: updateActiveJobCode() - Firestore 업데이트 (15% → 30%)
  ↓
Step 3: prefetchCampData() - 새 캠프 데이터 프리페칭 (30% → 60%)
  ↓
Step 4: WebView 프리로딩 대기 (60% → 100%)
  ↓
완료 알림 표시
```

### 3️⃣ 앱 재시작 (이미 로그인된 경우)
```
앱 실행
  ↓
CampTabContext: loadLastPreloadLinks()
  ├── AsyncStorage에서 마지막 프리로드 링크 복원
  └── preloadLinks 상태만 업데이트 (프리로드 시작 X)
  ↓
AuthContext: onAuthStateChanged
  ↓
[1️⃣ 앱 시작 플로우와 동일]
```

6. 페이드아웃 후 메인 화면

1. 마이페이지에서 다른 캠프 선택

3. 4단계 진행률 표시:
   - 📦 기존 캐시 정리 중... (0-15%)

4. 메인 화면 즉시 표시

## 🚀 성능 최적화

### 1. 중복 방지
- `hasPrefetchedRef`로 로그인 후 한 번만 프리페칭 실행
- `completedLinksRef`로 WebView 중복 로드 방지

### 2. 병렬 처리
- 모든 데이터 프리페칭을 `Promise.all()`로 병렬 실행
- WebView 프리로드도 동시에 진행

### 3. 캐싱
- React Query의 `staleTime`, `cacheTime` 활용
- AsyncStorage에 마지막 프리로드 링크 저장
- WebView 캐시 활성화 (`cacheEnabled: true`)

### 4. 선택적 프리로드
- 구글 시트 문서만 WebView 프리로드 (노션 제외)
- 활성화된 캠프의 데이터만 프리페칭
- 비로그인 상태에서는 프리로드 스킵

## 📊 프리로드 데이터 크기 예상

### 평균 프리로드 항목 (27기 멘토 기준)
- 수업 자료: 3-5개
- 시간표 링크: 2-3개
- 인솔표 링크: 1-2개
- 교육 자료 링크: 5-10개
- 반명단 학생: 50-100명
- 방명단 학생: 50-100명
- **총 WebView 프리로드: 약 10-20개**

### 예상 로딩 시간
- 데이터 프리페칭: 1-2초
- WebView 프리로드: 3-5초
- **총 시간: 약 4-7초**

## ⚠️ 주의사항

### 1. 네트워크 연결
- Wi-Fi 환경에서 최적 성능
- 모바일 데이터 사용량 증가 가능 (약 5-10MB)
- 네트워크 오류 시 에러 처리 (스플래시 1초 후 자동 닫기)

### 2. 앱 초기 로딩
- 첫 로그인 시 프리로드 시간이 길 수 있음
- 사용자에게 진행 상황을 명확히 표시하여 불안감 해소

### 3. 메모리 사용
- WebView 프리로드 시 메모리 사용량 증가
- 백그라운드에서 보이지 않게 처리 (opacity: 0.01)
