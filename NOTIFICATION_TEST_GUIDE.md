# 푸시 알림 테스트 가이드

## 테스트 화면 접근 방법

1. 앱 실행 후 로그인
2. 하단 탭에서 **마이페이지** 선택
3. **푸시 알림 테스트** 버튼 클릭

## 테스트 기능

### 1. 즉시 알림 (🚀)
- **기능**: 버튼을 누르면 즉시 알림 표시
- **테스트 방법**:
  1. 테스트 메시지 입력 (선택)
  2. "즉시 알림 전송" 버튼 클릭
  3. 알림이 즉시 표시됨

### 2. 예약 알림 (⏰)
- **기능**: 지정한 시간 후에 알림 표시
- **테스트 방법**:
  1. 초 단위 입력 (기본: 5초)
  2. "예약 알림 전송" 버튼 클릭
  3. **중요**: 앱을 백그라운드로 전환
  4. 지정한 시간 후 알림 수신

### 3. 업무 독촉 스타일 (📌)
- **기능**: 실제 업무 독촉 알림과 동일한 형식
- **테스트 방법**:
  1. "업무 알림 테스트" 버튼 클릭
  2. 앱을 백그라운드로 전환
  3. 5초 후 "📌 업무 알림" 스타일로 수신

### 4. 유틸리티 (🛠️)

#### 알림 권한 확인
- 현재 알림 권한 상태 확인
- iOS/Android 세부 권한 정보 표시

#### 푸시 토큰 확인
- Firestore에 저장된 푸시 토큰 확인
- 토큰이 정상적으로 등록되었는지 검증

#### 예약 알림 모두 취소
- 대기 중인 모든 예약 알림 취소

## 테스트 시나리오

### 시나리오 1: 기본 알림 작동 확인
```
1. 즉시 알림 전송 → 알림이 즉시 표시되는지 확인
2. 예약 알림 전송 (5초) → 백그라운드 전환 → 5초 후 알림 수신 확인
```

### 시나리오 2: 알림 권한 검증
```
1. 알림 권한 확인 클릭
2. "허용됨 ✅" 상태인지 확인
3. 만약 거부됨이면 → 기기 설정에서 권한 허용
```

### 시나리오 3: 푸시 토큰 검증
```
1. 푸시 토큰 확인 클릭
2. ExponentPushToken[...] 형식의 토큰 표시 확인
3. 토큰이 없으면 → 앱 재시작 후 다시 확인
```

### 시나리오 4: 실제 업무 독촉 알림 테스트
```
1. 업무 알림 테스트 클릭
2. 앱을 백그라운드로 전환
3. 5초 후 "📌 업무 알림" 수신 확인
4. 알림 클릭 → 앱이 열리는지 확인
```

## 주의사항

### ⚠️ 실제 기기에서만 테스트 가능
- **iOS 시뮬레이터**: 푸시 알림 미지원
- **Android 에뮬레이터**: 푸시 알림 미지원
- **반드시 실제 기기 사용**

### ⚠️ 알림 권한 필요
- 앱 최초 실행 시 알림 권한 요청
- 거부한 경우 → 기기 설정에서 수동 허용 필요

### ⚠️ 백그라운드 전환 필수
- 예약 알림 테스트 시 앱을 백그라운드로 전환해야 함
- 포그라운드 상태에서는 알림이 다르게 표시됨

## 문제 해결

### 알림이 수신되지 않을 때

#### 1. 권한 확인
```
테스트 화면 → 알림 권한 확인 → "허용됨 ✅" 인지 확인
```

#### 2. 푸시 토큰 확인
```
테스트 화면 → 푸시 토큰 확인 → 토큰이 등록되어 있는지 확인
```

#### 3. 실제 기기 사용 확인
```
시뮬레이터가 아닌 실제 iOS/Android 기기에서 테스트
```

#### 4. 앱 재시작
```
앱을 완전히 종료 후 다시 실행 → 자동으로 푸시 토큰 재등록
```

### 에러 메시지별 해결 방법

#### "푸시 토큰이 없습니다"
- 앱 재시작
- 로그아웃 후 다시 로그인
- 알림 권한 재확인

#### "알림 권한이 거부되었습니다"
- 기기 설정 → 앱 → 알림 → 허용

#### "Device가 아닙니다"
- 시뮬레이터가 아닌 실제 기기에서 테스트

## 실제 업무 독촉 알림 테스트

### 수동 테스트 방법

1. **업무 생성**
   - 캠프 탭 → 업무 탭
   - 새 업무 추가 (현재 시간보다 이전 시간으로 설정)

2. **Cloud Function 수동 실행**
   ```bash
   # Firebase Console → Functions → checkOverdueTasks
   # "테스트" 버튼 클릭
   ```

3. **30분 대기**
   - 자동 스케줄러는 30분마다 실행
   - 실제 환경에서는 자동으로 독촉 알림 전송

### 로그 확인

```bash
# Firebase Console → Functions → 로그
# 또는 터미널에서
npm run logs

# 확인할 내용:
# - "업무 독촉 알림 체크 시작"
# - "독촉 알림 전송 중"
# - "알림 전송 완료"
```

## 개발자 도구

### 콘솔 로그
앱 실행 중 다음 로그 확인:
```
✅ Firebase Authentication 계정 생성 완료
✅ 푸시 토큰 저장 완료: ExponentPushToken[...]
📱 알림 수신: {...}
👆 알림 응답: {...}
```

### Firestore 확인
```
Firebase Console → Firestore Database → users/{userId}

확인 항목:
- pushTokens: { [token]: { platform, addedAt, lastUsed } }
- notificationSettings: { taskReminders: true, generalNotifications: true }
```

## 다음 단계

1. ✅ 로컬 알림 테스트 완료
2. ✅ 푸시 토큰 등록 확인
3. ✅ 권한 상태 확인
4. ⏳ Cloud Functions 배포 후 실제 독촉 알림 테스트
5. ⏳ 프로덕션 환경에서 최종 검증

## 참고 사항

- 테스트 화면은 개발용이므로 프로덕션 배포 전 제거 또는 관리자 전용으로 제한 가능
- 모든 알림은 Expo Push Token을 통해 전송됨
- 실제 독촉 알림은 Cloud Functions에서 자동으로 전송됨
