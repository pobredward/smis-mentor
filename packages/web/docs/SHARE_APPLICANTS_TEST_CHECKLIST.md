## 기능 테스트

### Phase 1: 링크 생성 테스트

#### 1.1 관리자 페이지 접근
- [ ] `/admin/job-board-manage/applicants/[id]` 페이지 접근
- [ ] "현재 목록 공유" 버튼 표시 확인
- [ ] 필터링된 지원자 수 표시 확인

#### 1.2 모달 열기
- [ ] "현재 목록 공유" 버튼 클릭
- [ ] 모달 정상 표시 확인
- [ ] 캠프 정보 표시 확인
- [ ] 선택된 지원자 수 표시 확인

#### 1.3 유효 시간 설정
- [ ] 기본 옵션 버튼 (1시간, 6시간, 12시간, 24시간, 48시간, 72시간, 168시간) 작동 확인
- [ ] 직접 입력 필드 작동 확인
- [ ] 1-720시간 범위 제한 확인

#### 1.4 링크 생성
- [ ] "링크 생성" 버튼 클릭
- [ ] 로딩 상태 표시 확인
- [ ] 생성 성공 메시지 표시 확인
- [ ] 생성된 URL 형식 확인: `https://www.smis-mentor.com/shared/applicants/[64자리 토큰]`
- [ ] 만료 시간 표시 확인

#### 1.5 링크 복사 및 공유
- [ ] "복사" 버튼 클릭
- [ ] 클립보드에 링크 복사 확인
- [ ] 복사 완료 피드백 (버튼 색상 변경, "복사됨" 텍스트) 확인
- [ ] "새 탭에서 열기" 버튼 작동 확인

### Phase 2: 공유 페이지 접근 테스트

#### 2.1 유효한 링크 접근
- [ ] 생성된 링크로 접근
- [ ] 로그인 없이 페이지 접근 가능 확인
- [ ] 로딩 상태 표시 확인

#### 2.2 페이지 내용 확인
- [ ] 캠프 정보 (제목, 기수, 코드) 표시 확인
- [ ] 만료 시간 및 남은 시간 표시 확인
- [ ] 임시 공유 링크 안내 메시지 표시 확인
- [ ] 지원자 목록 표시 확인
- [ ] 지원자 수 표시 확인

#### 2.3 지원자 정보 확인
각 지원자별로 다음 정보가 올바르게 표시되는지 확인:
- [ ] 이름
- [ ] 지원 상태 (서류/면접/최종)
- [ ] 이메일
- [ ] 전화번호
- [ ] 지원일
- [ ] 학교 정보 (있는 경우)
- [ ] 평가 점수 요약 (있는 경우)
- [ ] 면접 일정 (있는 경우)
- [ ] 지원 경로 (있는 경우)

#### 2.4 반응형 디자인
- [ ] 데스크톱 레이아웃 확인
- [ ] 태블릿 레이아웃 확인
- [ ] 모바일 레이아웃 확인
- [ ] 스크롤 작동 확인

### Phase 3: 만료 처리 테스트

#### 3.1 만료된 링크 접근
- [ ] 짧은 유효 시간(1시간)으로 링크 생성
- [ ] 시간 경과 후 링크 접근
- [ ] 만료 에러 메시지 표시 확인 (410 Gone)
- [ ] "홈으로 돌아가기" 버튼 작동 확인

#### 3.2 자동 정리 API 테스트
- [ ] `/api/share-applicants/cleanup-expired` 수동 호출
- [ ] 만료된 토큰 비활성화 확인
- [ ] 응답에 비활성화된 개수 표시 확인
- [ ] Firestore에서 `isActive: false` 상태 확인

#### 3.3 Cron Job 테스트
- [ ] Vercel Dashboard에서 Cron Job 실행 로그 확인
- [ ] 6시간마다 자동 실행 확인
- [ ] 실행 성공 상태 확인

### Phase 4: 에러 처리 테스트

#### 4.1 잘못된 토큰
- [ ] 존재하지 않는 토큰으로 접근
- [ ] 404 에러 메시지 표시 확인

#### 4.2 비활성화된 토큰
- [ ] 수동으로 `isActive: false` 설정
- [ ] 접근 시 유효하지 않은 링크 메시지 확인

#### 4.3 API 에러
- [ ] 네트워크 에러 시나리오 테스트
- [ ] 에러 메시지 표시 확인

#### 4.4 지원자 없음
- [ ] 지원자가 삭제된 경우 처리 확인
- [ ] JobBoard가 삭제된 경우 처리 확인

### Phase 5: 보안 테스트

#### 5.1 인증 우회 시도
- [ ] 로그인 없이 관리자 API 호출 시도

#### 5.2 권한 에러
- [ ] 일반 사용자가 링크 생성 시도

#### 5.3 토큰 조작
- [ ] 토큰 문자열 변조 시도

#### 5.4 SQL Injection / XSS
- [ ] 특수 문자 입력 시도

#### 6.1 대량 지원자

## 문제 발생 시 대응

### 링크가 작동하지 않는 경우
1. Firestore에서 토큰 확인
2. 만료 시간 확인
3. `isActive` 상태 확인
4. API 로그 확인

### Cron Job이 실행되지 않는 경우
1. Vercel Dashboard에서 Cron Job 활성화 확인
2. `CRON_SECRET` 환경 변수 확인
3. API 엔드포인트 접근 가능 여부 확인
4. 수동 API 호출로 테스트

### 성능 문제
1. 한 번에 공유하는 지원자 수 제한
2. 데이터베이스 인덱스 추가
3. 페이지네이션 구현 (필요시)
4. 캐싱 전략 검토
